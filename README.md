## About Me

I enjoy building software around problems I personally find interesting or useful — especially when the work sits at the intersection of **engineering, mathematics, and programming**.

My main interests are:

* Building practical tools that solve real problems
* Developing data visualization and analysis applications
* Experimenting with machine learning models
* Understanding systems deeply, from algorithms to backend APIs
* Turning academic or day-to-day problems into usable software

I mainly work with **Python**, **Go**, and **JavaScript / TypeScript**, and I like projects that combine technical depth with clean, understandable implementation.

## Featured Projects

### Audio Signal Shift Correlation Visualization Tool

A Python desktop application for visualizing audio signal cross-correlation with user-defined time shifts.

The project focuses on efficient signal processing and real-time visualization, using FFT-based correlation methods to make cross-correlation analysis faster and more interactive.

**Tech focus:** Python, SciPy, signal processing, desktop GUI, data visualization

[View Repository](https://github.com/OmarElgaafary/Audio_Signal_Correlation_Shifting_Tool)

---

### Task Duration Estimator

An ongoing machine learning project that estimates the duration range of software tasks and JIRA-style tickets.

The model is trained on a large cleaned dataset of software tasks and uses separate preprocessing pipelines for text, categorical, and numerical features. The project also includes backend work for serving predictions through an API.

**Tech focus:** Python, machine learning, TF-IDF, Logistic Regression, FastAPI, Docker, model deployment

[View Repository](#)

### Backend & API Experiments

I also build backend projects to better understand how software systems are structured, including REST APIs, authentication flows, database-backed services, and small command-line utilities.
